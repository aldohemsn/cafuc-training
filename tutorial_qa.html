SCENARIO #4

火眼金睛：基于代码的智能质检

🚨 审校痛点

外部供应商交付了 50 页技术文档。人工肉眼检查格式错误（如中英文间距、标点混用）不仅低效，而且极易漏网。
作为数智化译员，我们拒绝使用笨重的办公软件，而是用代码和规则来从底层“清洗”和“质检”文本。

策略 A：正则排雷 (Regex in Sublime Text / Bash)

利用正则表达式 (Regular Expressions) 进行高精度的“模式匹配”，瞬间发现排版硬伤。这些操作可以在 Sublime Text 中完成，也可以直接在 Linux 终端用 grep 扫描。

bash — grep check 
# 1. 查找所有“中文字符”与“英文字母/数字”直接相连（缺少空格）的地方
grep -n -P "[\u4e00-\u9fa5][a-zA-Z0-9]|[a-zA-Z0-9][\u4e00-\u9fa5]" translation.txt

# 2. 查找中文里误用的英文标点（如中文句子里的英文逗号）
grep -n -P "[\u4e00-\u9fa5],|[\u4e00-\u9fa5]\." translation.txt

# 3. 查找数字后面没有单位，或者单位格式错误的行
# (示例：匹配数字后紧跟非空格字符的情况)
grep -n -P "\d+(?![\s\.\,])" translation.txt

注：以上正则通式同样适用于 Python 脚本或 Sublime Text 的查找栏（Ctrl+F > 开启 Regex 模式）。

策略 B：Python 自动化术语哨兵

编写一个轻量级的 Python 脚本，加载我们在 Stage 5 构建的 JSON 术语库，自动扫描供应商译文是否遵守了术语规范。

# 1. 加载术语库 (JSON 格式)

策略 C：Gemini 模型语义取证

代码检查格式，AI 检查灵魂。利用大模型识别“机翻痕迹”（Translationese）和逻辑漏洞。

Prompt Engineering (指令工程)

"你是一名资深的航空技术文档审校。请阅读下面的中文译文，分析其是否具有'机翻痕迹'。不要关注语法错误，重点关注：
1. 名词堆砌（过度使用'的'）
2. 被动语态滥用（xx被xx）
3. 不符合中文工程表述习惯的直译。
如果有，请指出并提供修正建议。"

将此 Prompt 集成到 Python 脚本中调用 Gemini API，即可实现对长文档的批量语义质检。

💡 讲师总结

真正的技术流译员，不依赖黑盒子的商业软件。我们用 Regex 解决 80% 的格式问题，用 Python 解决 100% 的术语一致性问题，最后用 AI 解决最难的语义风格问题。这就是数智化时代的“全栈审校”。
