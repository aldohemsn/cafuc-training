智能质检：为译文加上"安全锁"

在大型翻译项目中，术语一致性 (Consistency) 是质量控制的核心。 传统的肉眼检查容易疲劳且易出错。本模块将演示如何利用简单的 Python 脚本，打造一个不知疲倦的"术语哨兵"， 确保所有的关键概念（如联合国术语、法律专有名词）在数万字的文稿中保持绝对统一。

🎯 核心术语表 (示例)

英文术语 (Source) 标准译文 (Target) 来源/依据
Sustainable Development 可持续发展 UN / 联合国

Artificial Intelligence 人工智能 (非"AI") Style Guide

*注：在某些正式公文中，要求将 Artificial Intelligence 完整译为"人工智能"，而不能简写为 AI。

# 简单的术语自动检查脚本
# 用途：检查译文中是否包含了强制使用的标准术语

def check_terminology(source_text, target_text):
    # 定义标准术语库 (Dictionary)
    # Key: 英文原文关键词, Value: 必须出现的中文译法
    term_base = {
        "sustainable development": "可持续发展",
        "due diligence": "尽职调查",
        "artificial intelligence": "人工智能"
    }
    
    issues = []
    
    # 遍历术语库检查
    for eng, chn in term_base.items():
        if eng in source_text.lower():
            # 如果原文包含该术语，检查译文是否包含对应的标准译法
            if chn not in target_text:
                issues.append(f"❌ 错误: 原文包含 '{eng}'，但译文未找到标准译法 '{chn}'")
    
    return issues

# --- 测试案例 ---
src = "We must conduct due diligence on artificial intelligence projects."
tgt = "我们必须对AI项目进行调查。"  # 故意制造两个错误：简写了AI，漏译了尽职

print(f"原文: {src}")
print(f"译文: {tgt}\n")

errors = check_terminology(src, tgt)

if errors:
    print("⚠️ 发现术语一致性问题：")
    for err in errors:
        print(err)
else:
    print("✅ 无术语错误！")

思考题： 这个脚本目前只能做简单的字符串匹配。如果译文是"关于人工智能的尽职调查"，它能识别吗？
(答：能，因为包含子字符串。但如果中间加了形容词或断句，可能需要更复杂的正则表达式。)
