🔄 语料库建设核心流程

1

数据获取 (Data Acquisition)

目标：从《纽约时报》中文网获取"双语对照"版本的新闻文章。
方法：利用其 URL 结构（如 cn.nytimes.com/.../dual/）批量下载 HTML 页面。

技术要点：网页本质上是一个 HTML 文件。通过编程脚本（如 Python）可以自动下载数千个网页，远比手工复制粘贴高效。

2

数据清洗 (Data Cleaning)

目标：从 HTML 中剥离"纯净"的中英文对照段落，去除广告、导航、页脚等干扰信息。
方法：使用"DOM 解析"技术定位特定的 HTML 标签（如 <p class="paragraph">）。

技术要点：这里会用到 正则表达式 (Regex) 来处理特殊情况，如去除行首的页码 [1]、多余的换行符等。

3

数据结构化 (Structuring)

目标：将清洗后的文本转化为标准化的 JSON 格式，便于后续存储、检索和分析。

我们将得到类似下面的结构化数据：

{
    "metadata": {
        "url": "https://cn.nytimes.com/climate/20230728/.../dual/",
        "title_english": "This Looks Like Earth's Warmest Month...",
        "title_chinese": "这看起来是地球有记录以来最热的一个月",
        "date": "2023-07-28",
        "category": "climate"
    },
    "content": [
        {
            "sentence_english": "This month will likely go down as the hottest month...",
            "sentence_chinese": "这个月可能将成为有记录以来最热的月份..."
        }
        // ...更多句对
    ]
}
4

入库与应用 (Storage & Application)

目标：将结构化的 JSON 数据导入专业的"全文检索引擎"，实现毫秒级的双语检索。

后端：Elasticsearch / Solr

对 sentence_english 和 sentence_chinese 字段建立索引，支持关键词高亮、模糊匹配。

前端：Web 检索界面

开发一个简单的网页，输入关键词后显示中英对照结果，相当于"私有版 Google Translate 记忆库"。

Python 编程语言详解

一、Python 是什么？

Python 是一种通用编程语言。你可以用它来告诉计算机"做什么事情"、"按什么顺序做"、"在什么条件下做"。

它诞生于 1991 年，最初是为了让代码更容易阅读和编写。如今，它已成为 AI 时代最流行的编程语言，被广泛用于数据科学、机器学习、网络爬虫、自动化办公等领域。

二、为什么是 Python？它有什么特点？

1. 语法简洁，接近自然语言

相比其他语言（如 Java、C++），Python 代码更短、更易读。例如，打印一句话只需要写 print("Hello")。

2. 丰富的"库"生态

"库"就像预制的工具箱。Python 有海量的库，别人已经写好了复杂功能，你只需"拿来用"。比如用 requests 库下载网页，用 BeautifulSoup 库解析 HTML。

3. 跨平台运行

同样的 Python 代码可以在 Windows、Mac、Linux 上运行，不需要修改。

4. 社区活跃，资源丰富

遇到问题时，搜索引擎几乎总能找到答案。AI 助手（如 ChatGPT）也能很好地帮助你编写和解释 Python 代码。

三、Python 在语料库建设中的作用

在本例的《纽约时报》双语数据库项目中，Python 扮演了"总指挥"的角色，贯穿整个流程：

阶段 Python 的作用 常用的库
1. 数据获取 自动访问数千个网页 URL，下载 HTML 文件。 requests
2. 数据清洗 解析 HTML 结构，提取中英文段落；用正则表达式去除噪音。 BeautifulSoup, re (正则)
3. 数据结构化 将文本组织成 JSON 格式并保存为文件。 json (Python 内置)
4. 入库 将 JSON 数据批量导入搜索引擎。 elasticsearch-py

四、完整的 Python 爬虫实例

以下是爬取《纽约时报》双语文章并转化为 JSON 语料文件的完整、可运行代码：

🔧 运行前准备：需要安装依赖库，在终端运行：pip install beautifulsoup4 requests 

#!/usr/bin/env python3
"""
《纽约时报》中文网双语文章爬取脚本
适用于 /dual/ 格式的双语对照页面
"""

import requests
from bs4 import BeautifulSoup
import json
import re
from datetime import datetime

def fetch_nyt_bilingual(url: str) -> dict:
    """从纽约时报中文网的双语页面提取内容"""
    
    # 1. 下载网页
    headers = {
        'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/120.0.0.0'
    }
    response = requests.get(url, headers=headers, timeout=30)
    response.encoding = 'utf-8'
    
    # 2. 解析 HTML
    soup = BeautifulSoup(response.text, 'html.parser')
    
    # 3. 提取元数据
    # 中文标题: h1 (不带 .en-title 类)
    title_cn = ''
    cn_title_elem = soup.select_one('header h1:not(.en-title)')
    if cn_title_elem:
        title_cn = cn_title_elem.get_text().strip()
    
    # 英文标题: h1.en-title
    title_en = ''
    en_title_elem = soup.select_one('header h1.en-title')
    if en_title_elem:
        title_en = en_title_elem.get_text().strip()
    
    # 作者: .byline address
    author = ''
    author_elem = soup.select_one('.byline address')
    if author_elem:
        author = author_elem.get_text().strip()
    
    # 日期: 从 URL 提取 (格式: /YYYYMMDD/)
    date_match = re.search(r'/(\d{8})/', url)
    date_str = f"{date_match.group(1)[:4]}-{date_match.group(1)[4:6]}-{date_match.group(1)[6:8]}" if date_match else ''
    
    # 4. 提取正文 (双语段落交替出现: 英, 中, 英, 中...)
    paragraphs = soup.select('div.article-paragraph')
    pairs = []
    
    for i in range(0, len(paragraphs) - 1, 2):
        en_text = paragraphs[i].get_text().strip()
        cn_text = paragraphs[i + 1].get_text().strip()
        if en_text and cn_text:
            pairs.append({
                'sentence_english': en_text,
                'sentence_chinese': cn_text
            })
    
    # 5. 返回结构化数据
    return {
        'metadata': {
            'url': url,
            'title_english': title_en,
            'title_chinese': title_cn,
            'author': author,
            'date': date_str
        },
        'content': pairs
    }

# ===== 主程序 =====
if __name__ == '__main__':
    url = "https://cn.nytimes.com/usa/20251218/trump-immigration-citizenship-denaturalization/dual/"
    
    data = fetch_nyt_bilingual(url)
    
    # 打印预览
    print(f"标题: {data['metadata']['title_chinese']}")
    print(f"句对数量: {len(data['content'])}")
    
    # 保存为 JSON 文件
    filename = f"{data['metadata']['date']}_{data['metadata']['title_chinese'][:20]}.json"
    with open(filename, 'w', encoding='utf-8') as f:
        json.dump(data, f, ensure_ascii=False, indent=4)
    
    print(f"已保存: {filename}")

代码解读：

  • requests.get(url)：向指定网址发送 HTTP 请求，下载网页内容。
  • BeautifulSoup(response.text, 'html.parser')：将 HTML 代码解析成可操作的"树形结构"。
  • soup.select_one('header h1.en-title')：使用 CSS 选择器定位特定元素。
  • for i in range(0, len(paragraphs) - 1, 2)：以步长 2 遍历，因为英中段落是交替出现的。
  • json.dump(..., ensure_ascii=False)：保存 JSON 时保留中文字符（不转成 \uXXXX）。

JSON 数据格式详解

什么是 JSON？

JSON（JavaScript Object Notation）是一种轻量级的数据交换格式。它最初是为 JavaScript 设计的，但由于其简洁和通用，现在已被所有主流编程语言支持。

你可以把 JSON 理解为"带层次结构的电子表格"。与 Excel 表格只能表示"行×列"的二维数据不同，JSON 可以表示任意层级的嵌套关系。

JSON 的基本语法

  • 键值对 (Key-Value Pair)：数据以 "键": "值" 的形式组织。例如："name": "张三"。
  • 对象 (Object)：用花括号 {} 包裹的一组键值对。例如：{"name": "张三", "age": 30}。
  • 数组 (Array)：用方括号 [] 包裹的有序列表。例如：["苹果", "香蕉", "橙子"]。
  • 嵌套：对象和数组可以相互嵌套。例如，一个对象的值可以是另一个对象或数组。

实例解析：《纽约时报》双语数据

以下是一条来自真实语料的 JSON 记录（已简化）：

{
    "metadata": {
        "title_english": "This Looks Like Earth's Warmest Month.",
        "title_chinese": "这看起来是地球有记录以来最热的一个月",
        "date": "2023-07-28"
    },
    "content": [
        {
            "sentence_english": "Researchers said this week that the temperatures would have been 'virtually impossible' without human-driven climate change.",
            "sentence_chinese": "研究人员本周表示，如果没有人为造成的气候变化影响，这些地区在这么多天内观察到的气温'几乎是不可能的'。"
        }
    ]
}

解读：最外层是一个对象 {}，包含 metadata 和 content 两个键。metadata 的值又是一个对象（包含标题、日期）；content 的值是一个数组 []，内含多个"句对"对象。

正则表达式 (Regex) 详解

什么是正则表达式？

正则表达式（Regular Expression，简称 Regex）是一种用于描述文本模式的语法。它可以让你用一行简洁的"公式"来匹配、查找或替换符合特定规则的文本。

想象一下：你有一份 1000 页的文档，需要找出所有的电话号码。手动搜索太慢了。如果你知道电话号码的"模式"（如 11 位数字，以 1 开头），就可以用正则表达式一次性找出全部。

常用的正则符号

符号 含义 示例
. 匹配任意单个字符 a.c 匹配 "abc", "aXc"
* 前一个字符出现 0 次或多次 ab*c 匹配 "ac", "abc", "abbc"
+ 前一个字符出现 1 次或多次 ab+c 匹配 "abc", "abbc" (不匹配 "ac")
\d 匹配任意数字 (0-9) \d{11} 匹配 11 位数字
\s 匹配空白字符（空格、Tab、换行） Hello\sWorld 匹配 "Hello World"
[abc] 匹配方括号内的任意一个字符 [aeiou] 匹配任意元音字母
^ 匹配行首 ^Hello 匹配以 "Hello" 开头的行
$ 匹配行尾 end$ 匹配以 "end" 结尾的行

实战应用：数据清洗

在构建语料库时，从网页提取的文本往往包含"噪音"。正则表达式是处理这些问题的利器：

  • 去除页码：网页中可能有 [1], [2] 这样的分页标记。可以用 \[\d+\] 匹配并删除它们。
  • 合并换行：一个段落可能被错误地分成了多行。可以用 \n+ 匹配连续的换行符，替换为单个空格。
  • 提取日期：从 URL cn.nytimes.com/climate/20230728/... 中提取日期，可以用 \d{8} 匹配 20230728。

HTML 网页结构详解

什么是 HTML？

HTML（HyperText Markup Language，超文本标记语言）是所有网页的底层语言。当你打开任何网页时，你的浏览器实际上是在读取一个 HTML 文件，并把它"画"成你看到的页面。

HTML 使用"标签"（Tag）来定义页面的结构。标签成对出现，如 <p>这是一个段落</p>。

常见的 HTML 标签

标签 用途
<h1> - <h6> 标题（h1 最大，h6 最小）
<p> 段落 (Paragraph)
<a href="..."> 超链接 (Anchor)
<div> 通用容器（用于分组内容）
<span> 行内容器（用于标记文本片段）
<ul> / <ol> / <li> 无序/有序列表 及 列表项

属性 (Attributes)：标签的"附加信息"

标签可以有"属性"来提供额外信息。最常见的两个属性是：

  • class：用于分类标签，方便用 CSS 设置样式或用 JavaScript 操作。例如 <p class="paragraph_en"> 表示这是一个英文段落。
  • id：用于唯一标识某个标签。一个页面中，同一个 id 只能出现一次。

实战应用：从网页中提取数据

当我们要从《纽约时报》网页提取双语内容时，需要定位到包含文章内容的特定 HTML 标签。例如：

<div class="article-body">
    <p class="paragraph_en">This month will likely go down as the hottest...</p>
    <p class="paragraph_cn">这个月可能将成为有记录以来最热的月份...</p>
    ...
</div>

提取逻辑：用编程工具（如 Python 的 BeautifulSoup 库）查找所有 class="paragraph_en" 和 class="paragraph_cn" 的 <p> 标签，分别提取其中的文本，然后配对保存。
